Stop paying frontier-model prices for routine data extraction. Our scripts analyze your JSON payloads before they hit the LLM.

1. Right-Size the Model

Asking reasoning models to parse simple data is slow. We calculate object depth and tokens to recommend routing between fast models (Gemini Flash) and reasoning models (Gemini Pro).

Supported JSON Formats

The LLM Markdown Optimizer automatically detects the structure of your JSON and routes it to the most token-efficient Markdown representation. Here is how it handles different data shapes.

1. Flat Key-Value Pairs (The "Dictionary")

Used for a single entity or object. The tool converts this into a clean, bulleted list. It automatically normalizes camelCase keys and translates booleans/nulls into human-readable text.

Input JSON
{
  "user_id": 101,
  "firstName": "Ada",
  "isActive": true,
  "cancellation_date": null
}
Optimized Markdown
- **User Id**: 101
- **First Name**: Ada
- **Is Active**: Yes

*Notice how cancellation_date is dropped to save tokens, and isActive becomes "Yes".

2. Arrays of Uniform Objects (The "Database Table")

Used for lists of items that share similar keys. If left as JSON, keys repeat for every item, wasting tokens. The tool extracts keys into a single header row and generates a Markdown table.

Input JSON
[
  {"id": 1, "taskName": "Scrape Data", "status": "Done"},
  {"id": 2, "taskName": "Parse JSON", "status": "Pending"}
]
Optimized Markdown
| Id | Task Name | Status |
|---|---|---|
| 1 | Scrape Data | Done |
| 2 | Parse JSON | Pending |

3. Nested / Hierarchical Data (The "Tree")

Used for complex payloads. Creates structural boundaries using headings (##) and nested bullets. Crucial Feature: If it detects an array of objects inside nested data, it renders that specific section as a Markdown table.

Input JSON
{
  "project": "Migration",
  "details": {
    "team_size": 5,
    "tools": ["Python", "LLMs"]
  },
  "milestones": [
    {"phase": 1, "name": "Planning"},
    {"phase": 2, "name": "Execution"}
  ]
}
Optimized Markdown
- **Project**: Migration

## Details
- **Team Size**: 5
- **Tools**:
  - Python
  - LLMs

## Milestones (Data Table)
| Phase | Name |
|---|---|
| 1 | Planning |
| 2 | Execution |
